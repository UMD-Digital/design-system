Getting Started

This guide demonstrates how to use UMD Web Components in your HTML pages. All examples include the necessary script setup and show practical usage patterns.

Basic Setup

First, include the UMD Web Components library in your HTML page:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My UMD Page</title>
</head>
<body>
    <!-- Your content here -->
    <umd-element-card data-theme="light">
        <h3 slot="headline">Welcome to UMD</h3>
        <p slot="text">Discover fearless ideas.</p>
    </umd-element-card>
    
    <!-- 
    ⚠️ CDN Usage - For prototyping and development only!
    Production applications should use npm and a build process for:
    - Better performance (tree-shaking, bundling)
    - Version control and dependency management
    - Offline availability
    - No external dependencies on third-party CDNs
    -->
    <script src="https://unpkg.com/@universityofmaryland/web-components-library/dist/index.js"></script>
    
    <!-- Production Method (Recommended): Install via npm -->
    <!-- npm install @universityofmaryland/web-components-library -->
    <!-- Then use a build tool like Webpack, Vite, or Rollup -->
    <!--
    <script type="module">
        import LoadUmdComponents from '@universityofmaryland/web-components-library';
        LoadUmdComponents();
    </script>
    -->
</body>
</html>
Note: If using a CDN or bundler, adjust the import path accordingly. Components must be loaded before they can be used in the DOM.

Basic Component Usage

Card Component

The card component is one of the most versatile components for displaying content:

<umd-element-card data-theme="light">
    <img slot="image" src="campus-photo.jpg" alt="UMD Campus" />
    <p slot="eyebrow">News & Updates</p>
    <h3 slot="headline">Welcome to the University of Maryland</h3>
    <p slot="text">
        Discover our vibrant campus community and world-class academic programs.
        Join us in fearless ideas that change the world.
    </p>
    <div slot="actions">
        <a href="/learn-more" class="btn">Learn More</a>
    </div>
</umd-element-card>

Alert Component

Display important messages or notifications:

<umd-element-alert data-type="info">
    <h4 slot="headline">Campus Update</h4>
    <p slot="text">
        The library will have extended hours during finals week.
    </p>
</umd-element-alert>

Accordion Component

Create collapsible content sections:

<umd-element-accordion>
    <umd-element-accordion-item data-visual-open="true">
        <h3 slot="headline">Admissions Requirements</h3>
        <div slot="text">
            <p>Learn about our admissions criteria and application process.</p>
            <ul>
                <li>GPA requirements</li>
                <li>Standardized test scores</li>
                <li>Letters of recommendation</li>
            </ul>
        </div>
    </umd-element-accordion-item>
    
    <umd-element-accordion-item>
        <h3 slot="headline">Financial Aid</h3>
        <div slot="text">
            <p>Explore scholarship and financial aid opportunities.</p>
        </div>
    </umd-element-accordion-item>
</umd-element-accordion>

Using the Styles Package with Web Components

The UMD Styles Package works seamlessly with the Web Components Library to provide consistent spacing, colors, theming, and layout utilities. This combination ensures your components follow UMD brand guidelines while maintaining flexibility.

Installing the Styles Package

# Using npm
npm install @universityofmaryland/web-styles-library

# Using yarn
yarn add @universityofmaryland/web-styles-library

Layout Spacing Classes

The styles package provides predefined spacing classes that work perfectly with components:

<!-- Vertical spacing for landing pages -->
<div class="umd-layout-space-vertical-landing">
  <umd-element-hero type="default">
    <!-- Hero content -->
  </umd-element-hero>
</div>

<!-- Horizontal padding/margins -->
<div class="umd-layout-space-horizontal-large">
  <umd-element-card>
    <!-- Card content -->
  </umd-element-card>
</div>

<!-- Available spacing sizes -->
<!-- horizontal: extra-small, smaller, small, large, larger -->
<!-- vertical: landing, landing-child -->

Grid Layout Classes

Create responsive grids for component layouts:

<!-- Responsive grid with gap -->
<div class="umd-grid-gap">
  <umd-element-card><!-- Card 1 --></umd-element-card>
  <umd-element-card><!-- Card 2 --></umd-element-card>
  <umd-element-card><!-- Card 3 --></umd-element-card>
</div>

<!-- Masonry layout -->
<div class="umd-grid-gap-masonry umd-animation-grid">
  <umd-element-card-overlay><!-- Content --></umd-element-card-overlay>
  <umd-element-card-overlay><!-- Content --></umd-element-card-overlay>
</div>

<!-- Three column grid -->
<div class="umd-grid-gap-three">
  <umd-element-article><!-- Article 1 --></umd-element-article>
  <umd-element-article><!-- Article 2 --></umd-element-article>
  <umd-element-article><!-- Article 3 --></umd-element-article>
</div>

<!-- Stacked grid for mobile/tablet -->
<div class="umd-grid-gap-stacked">
  <umd-element-stat><!-- Stat 1 --></umd-element-stat>
  <umd-element-stat><!-- Stat 2 --></umd-element-stat>
</div>

Typography Classes

Apply consistent typography with style utilities:

<!-- Heading styles -->
<h2 class="umd-sans-largest-uppercase">By The Numbers</h2>

<!-- Rich text formatting -->
<div class="umd-text-rich-simple-large">
  <p>Large paragraph text with consistent styling.</p>
</div>

<!-- Dark theme text -->
<div class="umd-text-rich-simple-large-dark">
  <p>Text styled for dark backgrounds.</p>
</div>

<!-- Adjacent line text for special layouts -->
<h2 class="umd-adjacent-line-text">Research Enterprise</h2>

Background and Theme Classes

Apply themed backgrounds to sections:

<!-- Quarter-tone light background -->
<section class="umd-layout-background-quarter-light">
  <umd-element-section-intro data-theme="quarter">
    <!-- Content -->
  </umd-element-section-intro>
</section>

Animation Classes

Add entrance animations to component grids:

<div class="umd-animation-grid">
  <umd-element-card style="animation-delay: 0ms;">
    <!-- First card appears immediately -->
  </umd-element-card>
  
  <umd-element-card style="animation-delay: 350ms;">
    <!-- Second card appears after 350ms -->
  </umd-element-card>
  
  <umd-element-card style="animation-delay: 700ms;">
    <!-- Third card appears after 700ms -->
  </umd-element-card>
</div>

Sticky Columns Layout

Create advanced layouts with sticky positioning:

<umd-element-sticky-columns class="umd-layout-space-horizontal-large" 
  position-top="92px">
  <div slot="sticky-column">
    <!-- This content stays in place while scrolling -->
    <umd-element-card-overlay data-theme="dark">
      <!-- Card content -->
    </umd-element-card-overlay>
  </div>
  
  <div slot="static-column">
    <!-- This content scrolls normally -->
    <umd-feed-news show-count="3"></umd-feed-news>
  </div>
</umd-element-sticky-columns>
Best Practices
  • Use layout spacing classes for consistent margins and padding
  • Apply grid classes for responsive component layouts
  • Combine typography utilities with component slots for branded text
  • Use animation classes to create engaging entrance effects
  • Theme components and their containers together for cohesive design

Common Attributes

Configuration Attributes (data-*)

These attributes set the initial state and appearance of components:

Attribute Values Description
data-theme light, dark Component color scheme
data-display grid, list, inline Layout style
data-visual-open true, false Initial open/closed state
data-size small, medium, large Component size

Responsive Components

Components automatically adapt to different screen sizes. You can also use responsive attributes:

<umd-element-carousel data-columns="3" data-columns-tablet="2" data-columns-mobile="1">
    <!-- Carousel items -->
</umd-element-carousel>

Form Components

Search Component

<umd-element-search data-theme="light">
    <form slot="form" action="/search" method="get">
        <input type="search" name="q" placeholder="Search UMD..." />
        <button type="submit">Search</button>
    </form>
</umd-element-search>

Best Practices

Accessibility
  • Always provide meaningful text in slots, especially headlines
  • Use semantic HTML elements (h1-h6 for headlines, p for text)
  • Include alt text for all images
  • Ensure proper heading hierarchy
Performance
  • Load only the components you need for better performance
  • Lazy load images when possible
  • Components are automatically code-split and loaded on demand
Styling
  • Components use Shadow DOM - external CSS won't affect internal styles
  • Use theme attributes for consistent styling
  • Components follow UMD brand guidelines automatically
